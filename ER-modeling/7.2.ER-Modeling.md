# The E-R data model

- employs three basic concepts:
  - 1. entity sets,
  - 2. relationship sets, and
  - 3. attributes

## 7.2.1. Entity Set

- An entity may be **concrete**, such as a **person** or a **book**, or
- it may be **abstract**, such as a **course**, a **course offering**, or a **flight reservation**.
- An entity set is a set of entities of the same type that share the same properties, or attributes.
- Entity sets do not need to be disjoint
  - 2️⃣ Meaning of **“Disjoint”**
    - Two sets are disjoint if they do not share any common elements.
      ```txt
      - Example:
      - Set A = {1,2,3}
      - Set B = {4,5,6}
      - No common elements → Disjoint
      ```

## 7.2.2. Relationship Sets

A **relationship set** is a set of relationships of the same type. Formally, it is a mathematical relation on n ≥ 2(possibly non distinct)entity sets.

```txt
If E1, E2,...,En are entity sets, then a relationship set R is a subset of

{(e1, e2,...,en) | e1 ∈ E1,e2 ∈ E2,...,en ∈ En}

where (e1,e2,...,en) is a relationship.
```

-  Meaning of **"Possibly Non Distinct Entity Sets"**
-  
```txt
This is the important concept.

It means:

The same entity set can appear more than once in a relationship.

In other words, a relationship can connect entities from the same entity set.
```

This is called a **recursive relationship**.


``` txt
Employee supervises Employee
Person married to Person
Employee manages Employee
```

6️⃣ Why Roles Are Important

Roles help when:

1️⃣ Same entity set appears multiple times

Example:

Employee manages Employee

Roles:

Manager

Worker

------------------------------------------------------

In the Entity–Relationship (ER) model, the statement:

“The function that an entity plays in a relationship is called that entity’s role.”

means that an entity can participate in a relationship in a specific capacity or function, and that function is called its role.

1️⃣ Why do we need Roles?

Sometimes a relationship connects entities that belong to the same entity set or even different sets but with different meanings in the relationship.

To clarify who is doing what, we define roles.

A role describes the purpose of an entity within a relationship.
